Ward Directory — Thane Municipal Corporation

📊 Official Data Integration Guide

⚠ NCRB does not publish ward-level data. The finest official granularity is district/city level (Thane District, Maharashtra). Ward-level breakdown requires your own FIR data from the CCTNS system — which your project already collects via CrimeReport. The strategy below uses both.

Source 1 · NCRB Open Data (City-level baseline)

1
Go to data.gov.in NCRB Catalog
Visit data.gov.in/catalog/crime-india-2023
Click "District-wise crimes under IPC" → Download CSV.
Filter rows where STATE/UT = "Maharashtra" and DISTRICT = "Thane".
2
Extract Thane city totals (2018–2023)
The CSV has columns: Year, State, District, Murder, Robbery, Theft, Assault…
Run this Python script to extract and save:

import pandas as pd
df = pd.read_csv('district_ipc_crimes.csv')
thane = df[(df['District']=='Thane') & (df['State']=='Maharashtra')]
thane.to_csv('home/thane_ncrb_baseline.csv', index=False)
3
Map district totals to ward weights
Since NCRB has no ward breakdown, use your internal CrimeReport records to calculate each ward's share of total city crimes.
Then multiply NCRB district totals × ward share to get calibrated ward estimates.
Add a management command: python manage.py calibrate_ward_predictions
(Starter code is in home/management/commands/ — create it using the snippet below.)

Source 2 · Your Own CrimeReport DB (Ward-level, live)

4
Add ward field to CrimeReport model
In home/models.py, your CrimeReport already has address and latitude/longitude.
Add: ward = models.ForeignKey('Ward', null=True, blank=True, on_delete=models.SET_NULL)
Then in report_crime() view, after saving the report, do a point-in-polygon lookup using the thane_wards.json GeoJSON to auto-assign the ward.
5
Build the ward summary API endpoint
Create GET /api/ward-intel/<ward_name>/ that returns:
• Recent reports (last 30 days) for that ward
• Crime type breakdown from your DB
• Peak hour histogram (group reported_at__hour)
• Resolution rate for that ward
The template's JS already calls this endpoint — just wire it up in urls.py.
6
Update ward_predictions.json generation
Replace the current static JSON with a Django management command that:
1. Pulls NCRB baseline totals for Thane
2. Calculates ward shares from CrimeReport
3. Runs your existing ml_utils.predict_severity per ward
4. Writes out home/ward_predictions.json
Schedule it with django-crontab: run nightly at 2am.

Source 3 · Maharashtra Police Open Data

7
mahapolice.gov.in FIR data (PS-level)
Visit mahapolice.gov.in → Citizens' Corner → Crime Statistics.
Thane Police Commissionerate publishes police station-wise FIR counts annually (PDF/Excel).
Download, extract with tabula-py or camelot, and join to your ward→police-station mapping.
8
RTI as last resort
If online data is insufficient, file an RTI via rtionline.gov.in to Thane CP Office requesting ward/PS-wise IPC crime data for 2018–2024.
Response time: 30 days. Cost: ₹10. This is the most granular official data possible.
💡 Bottom line: Your current ward_predictions.json is ML-generated from your own thane_crime_data.csv. That is legitimate and valuable — it reflects actual reports in your system. The NCRB layer adds a calibration baseline. The combination of both (internal live data + NCRB city baseline) is actually more granular than NCRB alone.
